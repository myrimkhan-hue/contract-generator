# Генератор договоров

Веб-приложение для автоматической генерации договоров перевозки грузов.

## Что внутри

- **Фронтенд** — простой HTML/CSS/JS интерфейс (без сборки, без фреймворков)
- **Бэкенд** — Node.js + Express, подставляет реквизиты в шаблон `template.docx`
- **3 ваши компании** уже зашиты в коде сервера (`server.js`)
- **История** — последние 20 договоров хранятся на сервере в `history.json` + копии .docx в папке `archive/`
- **Автонумерация** в формате `ГГ-ММ-ДД-XXX` (XXX — порядковый номер в рамках дня)

## Как запустить локально (Windows / macOS / Linux)

1. Установите [Node.js](https://nodejs.org/) (версия 18 или новее).
2. Откройте терминал в папке с проектом и выполните:
   ```bash
   npm install
   npm start
   ```
3. Откройте в браузере: <http://localhost:3000>

## Как запустить на сервере

### Вариант A: VPS / выделенный сервер (Ubuntu)

```bash
# Установка Node.js
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# Запуск приложения
cd /opt/contract-app
npm install
npm start
```

Чтобы держать сервис постоянно работающим — используйте `pm2`:
```bash
sudo npm install -g pm2
pm2 start server.js --name contracts
pm2 startup
pm2 save
```

Поставьте перед приложением nginx с HTTPS — и можно открывать сотрудникам по внутренней ссылке.

### Вариант B: облачные платформы

Подойдёт любая платформа, поддерживающая Node.js:
- **Railway** / **Render** / **Fly.io** — деплой за 5 минут, бесплатные тарифы
- **Heroku** — стандартный Node.js buildpack
- **VPS** на любом провайдере (Hetzner, DigitalOcean, Selectel)

Порт берётся из переменной окружения `PORT`, если она задана.

## Как добавить/изменить компании

Откройте `server.js`, найдите блок `OUR_COMPANIES` (примерно в начале файла) и отредактируйте.
Каждая компания имеет такой набор полей:

```js
ava_solution: {
  short: 'AVA Solution',           // короткое имя для карточки выбора
  type: 'ТОО',                     // ТОО / ИП / АО
  name: 'ТОО "AVA Solution"',      // полное наименование в договоре
  bin: '180840002872',
  address: 'РК, г. Алматы, ...',
  account: 'KZ16...',
  bank: 'АО «Банк ЦентрКредит»',
  bik: 'KCJBKZKX',
  position: 'Генеральный директор',
  signerFull: 'Юань Вэн-Лун',
  signerShort: 'Юань В.',
  basis: 'Устава',                 // в родительном падеже!
  phone: '+7 ...',
  email: 'info@...',
},
```

Чтобы добавить 4-ю компанию — просто добавьте ещё один ключ в объекте `OUR_COMPANIES`.

## Как изменить шаблон договора

Файл `template.docx` — это обычный Word-документ. Откройте его в Word/LibreOffice, измените текст, **но не трогайте метки в фигурных скобках**:

| Метка | Значение |
|---|---|
| `{НОМЕР_ДОГОВОРА}` | Автоматический номер |
| `{ДАТА_ДОГОВОРА}` | Сегодняшняя дата в формате "20 мая 2026 г." |
| `{ЗАКАЗЧИК_НАЗВАНИЕ}` | Полное наименование заказчика |
| `{ЗАКАЗЧИК_БИН}` | БИН/ИИН заказчика |
| `{ЗАКАЗЧИК_АДРЕС}` | Юридический адрес |
| `{ЗАКАЗЧИК_СЧЕТ}` | Расчётный счёт (ИИК) |
| `{ЗАКАЗЧИК_БАНК}` | Банк |
| `{ЗАКАЗЧИК_БИК}` | БИК |
| `{ЗАКАЗЧИК_ДОЛЖНОСТЬ}` | Должность подписанта |
| `{ЗАКАЗЧИК_ПОДПИСАНТ}` | Полное ФИО подписанта |
| `{ЗАКАЗЧИК_ПОДПИСАНТ_КРАТКО}` | Фамилия и инициалы |
| `{ЗАКАЗЧИК_ОСНОВАНИЕ}` | Устава / Талона |
| `{ЗАКАЗЧИК_ТЕЛЕФОН}` | Телефон |
| `{ЗАКАЗЧИК_EMAIL}` | Email |

Те же метки есть для исполнителя — замените `ЗАКАЗЧИК` на `ИСПОЛНИТЕЛЬ`.

**Важно:** каждая метка должна быть в Word одним непрерывным куском текста. Если набирать прямо в Word, Word иногда разбивает текст на несколько внутренних фрагментов — тогда подстановка не сработает. Самый надёжный способ: скопируйте метку из таблицы выше и вставьте в нужное место как обычный текст (Ctrl+Shift+V → "сохранить только текст"). После правки сохраните файл как `.docx`.

## Безопасность

В текущем виде приложение не требует авторизации — любой, у кого есть ссылка, может генерировать договоры. Это нормально для внутренней сети. Если хотите ограничить доступ:

1. **Простой вариант** — Basic Auth через nginx:
   ```
   location / {
     auth_basic "Контракты";
     auth_basic_user_file /etc/nginx/.htpasswd;
     proxy_pass http://localhost:3000;
   }
   ```
2. **Полный вариант** — встроить логин в само приложение (если потребуется — могу добавить).
