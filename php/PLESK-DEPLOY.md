# Развёртывание PHP-версии на хостинге PS.kz (панель Plesk)

Это PHP-версия генератора — она работает на вашем текущем шаред-хостинге PS.kz
(Plesk + PHP), **без VPS и без доплат**. Ставится на поддомен, основной сайт
`ava-solution.kz` не затрагивается.

Содержимое папки `php/` — это и есть готовое приложение. Его нужно положить в
корень поддомена.

---

## ⚠️ 0. Сначала освободите место на диске

В панели сейчас **Диск: 107%** (5483 МБ из 5 ГБ) — место переполнено. Пока
не освободите хотя бы ~200–300 МБ, ничего не загрузится и приложение не сможет
сохранять документы. Варианты:

- удалить старые бэкапы/логи основного сайта (Файлы → почистить);
- удалить неиспользуемые письма/вложения в почте;
- либо расширить тариф по диску у PS.kz.

Само приложение весит мало (~250 КБ кода + шаблоны), но генерируемые документы
копятся (хранится до 50 последних, это ~5–10 МБ).

---

## 1. Создать поддомен

В Plesk → **Сайты и домены** → **Добавить субдомен**:

- Имя субдомена: `docs` (получится `docs.ava-solution.kz`; можно `dogovor` и т.п.)
- Корневой каталог документов: Plesk предложит папку, например
  `docs.ava-solution.kz` — запомните её.

Если DNS домена управляется в Plesk (обычно так и есть на PS.kz), запись для
поддомена создастся автоматически. Если DNS внешний — добавьте A-запись
`docs` → IP хостинга.

---

## 2. Выбрать PHP 8.x для поддомена (рекомендуется)

У основного сайта стоит PHP 7.4 (помечен «устаревшая опция»). Для поддомена
лучше переключить на PHP 8.1+:

Plesk → у поддомена → **Настройки PHP** → версия **8.1 / 8.2 / 8.3** → Применить.

> Код совместим и с 7.4, но на 8.x быстрее и безопаснее. Обязательно должно быть
> включено расширение **zip** (в Plesk обычно включено по умолчанию; если нет —
> Настройки PHP → extensions → zip).

---

## 3. Загрузить файлы приложения

Нужно положить **содержимое папки `php/`** (не саму папку, а то, что внутри) в
корневой каталог поддомена. Внутри должно оказаться:

```
docs.ava-solution.kz/
├── index.html
├── .htaccess
├── api/
│   └── index.php
├── lib/
│   ├── companies.php
│   ├── helpers.php
│   └── storage.php
└── templates/
    ├── template.docx
    └── template_zayavka.docx
```

Способы загрузки (любой на выбор):

**A. Через Git (в Plesk есть модуль Git):**
- Plesk → у поддомена → **Git** → добавить репозиторий
  `https://github.com/myrimkhan-hue/contract-generator.git`
- В настройках развёртывания укажите, что публиковать нужно подпапку `php/`
  в корень поддомена (Plesk → «Дополнительный путь развёртывания» или скопируйте
  файлы из `php/` в docroot после клонирования).

**B. Через Файловый менеджер / FTP:**
- Скачайте папку `php/` из репозитория (или клонируйте, зайдите в неё)
- Загрузите всё её содержимое в корень поддомена.
- Убедитесь, что скрытый файл **`.htaccess`** тоже загрузился (в файловом
  менеджере включите показ скрытых файлов).

---

## 4. Права на запись

Приложение само создаст папку `data/` (история, справочник контрагентов,
сгенерированные файлы) при первом запросе. На Plesk PHP работает от имени
владельца подписки, поэтому запись обычно разрешена сразу. Если увидите ошибку
записи — в файловом менеджере задайте папке `data/` права **755** (или создайте
её вручную рядом с `index.html` и дайте право на запись).

---

## 5. Включить SSL (https)

Plesk → у поддомена → **SSL/TLS-сертификаты** → **Let's Encrypt** → выпустить
для `docs.ava-solution.kz`. Тариф это позволяет (в списке услуг есть Let's
Encrypt). После выпуска включите «Перенаправление с http на https».

---

## 6. Проверка

Откройте **https://docs.ava-solution.kz** — должно открыться приложение.
Проверьте:
- выбор компании и роли, создание договора (скачается .docx);
- вкладку «Заявка»;
- что документ появляется в блоке «Последние документы» и скачивается повторно.

---

## Обслуживание

**Реквизиты ваших компаний** зашиты в `lib/companies.php` — редактируйте этот
файл (через файловый менеджер) и сохраняйте; изменения применяются сразу.

**Резервная копия данных** — вся история и контрагенты лежат в папке `data/`
(`history.json`, `contacts.json`, `files/`). Для бэкапа скачайте эту папку.

**Обновление кода** — если правки идут через GitHub, повторите загрузку файлов
(п. 3). Папку `data/` при этом не трогайте — она с вашими документами.

---

## Почему PHP, а не Node.js

На этом хостинге (Plesk, шаред) в «Инструментах разработчика» доступен только
PHP — модуля Node.js нет. Поэтому генератор портирован на PHP: логика та же,
шаблоны те же, результат идентичен Node-версии. Если в будущем возьмёте VPS —
можно вернуться к Node-версии (инструкция в `DEPLOY.md` в корне репозитория).
